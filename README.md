# 🚀 פרויקט אוטומציה 1: סנכרון וסינון לידים (Make)

**סטטוס:** הושלם בהצלחה

**מטרה עסקית:** יצירת מערכת אוטומטית לקליטה, סינון ועיבוד לידים בזמן אמת, תוך שמירה על איכות נתונים גבוהה (פורמט מספרי טלפון) ומתן התראה מיידית לצוות המכירות.

---

### 🛠️ ארכיטקטורה וכלים

| רכיב | כלי | תפקיד |
| :--- | :--- | :--- |
| **פלטפורמת אוטומציה** | Make (Integromat) | ניהול כל זרימת העבודה. |
| **קלט** | Webhooks (בדיקה באמצעות Postman) | קבלת נתוני לידים גולמיים (JSON). |
| **עיצוב נתונים** | מודול Phone Number | עיצוב מספר הטלפון לפורמט E.164 / International לשמירת אפס מוביל. |
| **אחסון נתונים** | Google Sheets | רישום ושמירת הלידים המאושרים. |
| **התראות** | Gmail | שליחת הודעת אימייל מיידית לצוות המכירות. |

### ⚙️ תרשים זרימה (Workflow)

הזרימה מתבצעת ברצף הבא:

1.  **Webhook (קלט):** מקבל נתוני ליד (JSON).
2.  **Filter (סינון):** בודק אם השדה `lead_source` שווה ל-'Ad Campaign X'. רק אם התנאי מתקיים, התרחיש ממשיך.
3.  **Phone Number (עיבוד):** מעצב את מספר הטלפון.
4.  **Google Sheets (שמירה):** מוסיף שורה לגיליון עם הנתונים המעובדים.
5.  **Gmail (התראה):** שולח אימייל אוטומטי לצוות עם פרטי הליד.

### 🔑 תובנות מרכזיות ופונקציות מתקדמות

* **טיפול באפס מוביל:** שימוש במודול `Phone Number` היה הכרחי כדי למנוע מ-Google Sheets להסיר את האפס המוביל מהמספר (בעיה נפוצה).
* **לוגיקה עסקית:** שימוש בפילטר על בסיס נתון ספציפי (מקור ליד) מדגים יכולת לטפל בדרישות עסקיות ספציפיות.
* **בדיקת קלט:** שימוש ב-Postman לבדיקה יזומה של ה-Webhook מבטיח יציבות המערכת.
